## 1. プロダクト概要

### コンセプト

> 「通知ゼロ、プレッシャーゼロ、でも見返したとき笑える、泣ける」

育児記録アプリではなく、**育児日記アプリ**。  
「記録しなきゃ」というタスクではなく、「書きたいから書く」という自然な感情から生まれる日記体験を提供する。

### アプリ名

**あのね。**

子どもに語りかけるように日記を書く、という世界観がそのままアプリ名になっている。句点（。）がひとつあることで「ちゃんと聞いてるよ」という温かさと、子どもへの手紙のような感覚を表現している。

### ターゲットユーザー

**プライマリ:** 0〜3歳の子どもを持つ母親（スマートフォンメイン）  
**セカンダリ:** パートナー・祖父母（閲覧・コメント）

### プラットフォーム

- **Phase 1:** Android（Google Play Store）
- **Phase 2以降:** iOS（App Store）

### マネタイズ

- **Phase 1:** 完全無料

## 2. 設計思想・UI/UX方針

### 基本方針

| 原則             | 内容                                                          |
| ---------------- | ------------------------------------------------------------- |
| プレッシャーゼロ | 通知機能は一切実装しない。書かなかった日があっても責めないUI  |
| 開いてすぐ書ける | アプリ起動後2タップ以内で記録開始できる                       |
| 見返して感動する | 「1年前の今日」「成長タイムライン」で過去が自然に浮かび上がる |
| シンプルを守る   | 1画面の情報量を最小限に。機能より体験を優先する               |

### 書かなかった日の扱い

- カレンダーの空白日に×マークや強調表示は一切しない
- 書いた日だけが静かに光る
- 「○日連続記録中！」などのゲーミフィケーションは使わない

## 5. 画面仕様

### 5-1. 認証画面

**目的:** アプリ初回起動 / 未ログイン時

**表示要素:**

- アプリロゴ・キャッチコピー
- Googleでログインボタン

**仕様:**

- ログイン後、子ども情報が未登録の場合は「子ども登録画面」へ遷移
- 登録済みの場合はホームへ遷移

---

### 5-2. 子ども登録画面（初回のみ）

**目的:** 日記の主人公となる子どもの情報を登録

**入力項目:**
| 項目 | 形式 | 必須 |
|------|------|------|
| 名前（ニックネーム可） | テキスト（最大20文字）| ✅ |
| 生年月日 | 日付ピッカー | ✅ |
| アイコン画像 | 写真選択（任意）| ❌ |

**仕様:**

- 登録後はホーム画面へ遷移
- 後から設定画面で変更可能

---

### 5-3. ホーム（日記一覧）

**目的:** 書いた日記を時系列で確認する

**表示要素:**

- 子どもの名前 + 生後〇日 / 〇歳〇ヶ月（ヘッダー常時表示）
- 「1年前の今日」カード（該当する場合、画面上部に表示）
- 日記カードリスト（新しい順）

**日記カードの表示項目:**

- 日付（例: 4月15日 火曜日）
- 生後〇日表示
- テキスト（最大3行で省略、タップで全文表示）
- サムネイル画像（写真がある場合）

**インタラクション:**

- カードタップ → 日記詳細モーダルへ
- 長押し → 削除確認ダイアログ
- 右上のボタン → 書くタブへの誘導（FAB）

---

### 5-4. 書く（新規作成）

**目的:** 今日の日記を書く。とにかく速く・気軽に

**表示要素:**

- 日付表示（今日の日付、変更不可）※後日編集は設定で許可
- 生後〇日表示
- テキスト入力エリア（プレースホルダー: 「今日はどんな日でしたか？」）
- 写真追加ボタン（1枚のみ）
- 保存ボタン

**テキスト仕様:**

- 最大文字数: 500文字
- 1行でも保存可能（ひとことでOK）
- キーボード表示時に写真ボタン・保存ボタンがキーボード上に追従

**写真仕様:**

- 1日1枚のみ
- 選択後、クライアント側で圧縮処理
  - 最大サイズ: 1280px（長辺）
  - 圧縮品質: 80%（JPEG）
  - 圧縮後の目安: 200〜400KB程度
- Supabase Storageへアップロード

**保存仕様:**

- 1日に1件のみ（同じ日の日記が存在する場合は「今日の日記を編集しますか？」と確認）
- 保存成功時: ホームへ遷移 + 静かなアニメーション（スプラッシュ演出なし）

---

### 5-5. 思い出（振り返り）

**目的:** 過去の日記を感情的に楽しむ

**表示要素（3セクション）:**

#### ① 1年前の今日

- 1年前の同日の日記を自動表示
- 写真 + テキスト + 当時の生後日数
- 該当なしの場合は非表示

#### ② 成長タイムライン

- 月別に区切られた日記リスト
- 「生後〇日〜〇日」という形でグルーピング
- 写真付き日記は画像を強調表示

#### ③ カレンダービュー

- 月カレンダー形式
- 日記を書いた日にのみ小さなドット（●）表示
- 日付タップで当日の日記詳細へ

---

### 5-6. 設定

**表示項目:**

- 子どもの情報（名前・生年月日・アイコン変更）
- 家族を招待（招待コード発行・表示）
- 招待コードで参加
- ログアウト
- アプリ情報（バージョン、プライバシーポリシー、利用規約）

---

### 5-7. 日記詳細モーダル

**表示要素:**

- 日付 + 生後〇日
- 写真（フルワイド表示）
- テキスト全文
- 編集ボタン / 削除ボタン

## 6. 家族共有機能

### 概要

パスもんと同じ招待コード方式。1つの子どもプロフィールを複数のユーザーで共有する。

### 招待フロー

```
招待する側:
設定 → 「家族を招待」→ 招待コード生成（6桁英数字）→ コードをコピー / 共有

招待される側:
設定 → 「招待コードで参加」→ コード入力 → 参加完了 → ホームへ
```

### 権限設計

| 操作           | オーナー         | 家族メンバー         |
| -------------- | ---------------- | -------------------- |
| 日記を書く     | ✅               | ✅                   |
| 日記を閲覧     | ✅               | ✅                   |
| 日記を削除     | ✅（自分の日記） | ✅（自分の日記のみ） |
| 子ども情報変更 | ✅               | ❌                   |
| 招待コード発行 | ✅               | ❌                   |
| メンバー削除   | ✅               | ❌                   |

### 仕様詳細

- 招待コードの有効期限: 24時間
- 1つの子どもプロフィールに参加できるメンバー: 最大5人
- 日記一覧では「誰が書いたか」をアバター or イニシャルで表示

## 7. データベース設計（Supabase）

### テーブル構成

#### `users`

```sql
id          uuid PRIMARY KEY  -- Supabase Auth の user id
name        text
avatar_url  text
created_at  timestamptz DEFAULT now()
```

#### `children`

```sql
id            uuid PRIMARY KEY DEFAULT gen_random_uuid()
owner_id      uuid REFERENCES users(id)
name          text NOT NULL
birthday      date NOT NULL
icon_url      text
created_at    timestamptz DEFAULT now()
```

#### `child_members`（共有メンバー管理）

```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
child_id    uuid REFERENCES children(id)
user_id     uuid REFERENCES users(id)
role        text DEFAULT 'member'  -- 'owner' | 'member'
joined_at   timestamptz DEFAULT now()
UNIQUE(child_id, user_id)
```

#### `invite_codes`

```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
child_id    uuid REFERENCES children(id)
code        text UNIQUE NOT NULL  -- 6桁英数字
created_by  uuid REFERENCES users(id)
expires_at  timestamptz NOT NULL  -- 作成から24時間
used        boolean DEFAULT false
created_at  timestamptz DEFAULT now()
```

#### `diary_entries`

```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
child_id    uuid REFERENCES children(id)
author_id   uuid REFERENCES users(id)
entry_date  date NOT NULL
text        text  -- 最大500文字
photo_url   text  -- Supabase Storage のパス
created_at  timestamptz DEFAULT now()
updated_at  timestamptz DEFAULT now()
UNIQUE(child_id, author_id, entry_date)  -- 1人につき1日1件制約
```

### Row Level Security（RLS）方針

- `diary_entries` の読み書きは `child_members` に所属するユーザーのみ許可
- `children` の更新は `owner` ロールのみ許可

## 8. 画像処理仕様

### クライアント側圧縮（expo-image-manipulator）

```typescript
// 圧縮設定
const compressImage = async (uri: string) => {
  const result = await ImageManipulator.manipulateAsync(
    uri,
    [{ resize: { width: 1280 } }], // 長辺1280px
    { compress: 0.8, format: ImageManipulator.SaveFormat.JPEG },
  );
  return result;
};
```

### Supabase Storage 構成

```
bucket: diary-photos
path:  {child_id}/{entry_date}/{uuid}.jpg
```

- バケットはプライベート設定
- 署名付きURL（Signed URL）で表示（有効期限1時間）

## 10. Play Store 申請に向けた準備

### 必要なもの

| 項目                     | 内容                                   |
| ------------------------ | -------------------------------------- |
| アプリアイコン           | 512×512px（PNG）                       |
| スクリーンショット       | 最低2枚（推奨8枚）                     |
| フィーチャーグラフィック | 1024×500px                             |
| 短い説明文               | 最大80文字                             |
| 詳細説明文               | 最大4000文字                           |
| プライバシーポリシーURL  | 必須（子ども関連アプリのため特に重要） |
| コンテンツレーティング   | アンケート回答が必要                   |

### プライバシーポリシーの注意点

子どもの写真・情報を扱うため、以下を明記する必要あり：

- 収集するデータの種類
- データの保存場所（Supabase / GCP）
- 第三者提供の有無
- データ削除方法
