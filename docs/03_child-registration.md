# 03: 子ども登録画面

## 概要

初回ログイン後に表示される子ども情報の登録画面。日記の主人公となる子どものプロフィールを作成する。

## 背景

日記機能はすべて `child_id` に紐づくため、子ども登録が完了しないとアプリの主要機能を利用できない。認証直後のオンボーディングフローの一部。

## 対応内容

### 画面実装

- [x] `app/child-register.tsx` 作成
- [x] 名前入力（テキスト、最大20文字、必須）
- [x] 生年月日入力（日付ピッカー、必須）
- [ ] アイコン画像選択（任意、カメラロールから選択）

### データ保存

- [x] `children` テーブルにレコード作成
- [x] `child_members` テーブルに `role: 'owner'` で自分を追加
- [ ] アイコン画像がある場合は Supabase Storage にアップロード

### ナビゲーション

- [x] 認証後、`children` / `child_members` にレコードがない場合はこの画面へ遷移
- [x] 登録完了後、ホーム画面（タブ）へ遷移
- [x] `app/_layout.tsx` のルート保護に子ども登録チェックを追加
